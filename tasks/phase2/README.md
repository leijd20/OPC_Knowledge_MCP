## 任务列表

### 任务 2.1: rmcp 协议集成 🔴 高优先级

**状态**：⬜ 未开始  
**预计工作量**：2-3 天  
**负责人**：待分配

#### 目标

将当前自定义 JSON over HTTP 接口替换为标准的 MCP Streamable HTTP 协议。

#### 背景

当前实现：
- `POST /mcp` 接受自定义 JSON 格式
- 直接调用 `McpServer::handle_tool()`
- 不符合 MCP 协议规范

目标实现：
- 使用 rmcp 的 `StreamableHttpService`
- 实现标准 MCP 协议端点
- 支持工具发现（`tools/list`）
- 支持 JSON-RPC 2.0 格式

#### 子任务

- [ ] **2.1.1 研究 rmcp API**
  - 阅读 rmcp 文档和示例
  - 理解 `StreamableHttpService` 的使用方式
  - 理解 `ServerHandler` trait 的要求
  - 预计时间：4 小时

- [ ] **2.1.2 实现 ServerHandler trait**
  - 在 `src/mcp/server.rs` 中实现 trait
  - 将现有的 `handle_tool()` 逻辑适配到 trait 方法
  - 实现工具列表返回（`list_tools()`）
  - 预计时间：8 小时

- [ ] **2.1.3 更新 HTTP 路由**
  - 在 `src/http/mod.rs` 中集成 `StreamableHttpService`
  - 更新路由配置
  - 保持认证中间件的兼容性
  - 预计时间：4 小时

- [ ] **2.1.4 定义工具 Schema**
  - 为每个工具定义 JSON Schema
  - 包含参数类型、描述、默认值
  - 符合 MCP 规范
  - 预计时间：4 小时

- [ ] **2.1.5 测试 MCP 协议**
  - 使用 MCP 客户端测试连接
  - 验证工具发现功能
  - 验证工具调用
  - 验证错误处理
  - 预计时间：4 小时

#### 涉及文件

- `src/mcp/server.rs` - 实现 ServerHandler
- `src/mcp/tools.rs` - 添加 Schema 定义
- `src/http/mod.rs` - 集成 StreamableHttpService
- `Cargo.toml` - 确认 rmcp 依赖配置

#### 验收标准

- [ ] 可以使用标准 MCP 客户端连接
- [ ] 工具发现功能正常
- [ ] 所有 4 个工具可以正常调用
- [ ] 错误响应符合 MCP 规范
- [ ] 认证机制仍然有效

#### 参考资料

- [rmcp crates.io](https://crates.io/crates/rmcp)
- [rmcp docs.rs](https://docs.rs/rmcp)
- [MCP 协议规范](https://modelcontextprotocol.io/)
